# Content Publisher — 多平台适配发布

一篇文章，适配全网格式，最大化传播效果。

## 平台适配规则

### 微信公众号
- **段落**：每段不超过3行（手机屏幕）；大量留白
- **开头**：前2句必须是强钩子；读者滑动很快
- **格式**：用引用块突出关键洞察；加粗强调重点
- **图片**：1张封面 + 2-3张内嵌截图；不支持Mermaid（用文字描述流程）
- **结尾**：明确的CTA（关注、评论、试试看）
- **字数**：1500-2500字是甜蜜点

### 知乎
- **开头**：以"谢邀"或直接回答隐含问题开始
- **语调**：更多技术深度，假设读者想理解"为什么"
- **图表**：支持Mermaid，保留
- **链接**：尽可能添加参考链接和引用
- **标签**：文末使用相关话题标签
- **字数**：2000-4000字；有干货可以更长

### 掘金
- **开头**：包含"前言"说明读者将学到什么
- **代码**：包含可运行的代码示例，带语法高亮
- **图表**：支持Mermaid，保留
- **标签**：添加3-5个相关标签
- **字数**：1500-3000字；开发者快速扫读找可执行代码

### 小红书
- **字数**：推荐300字以内（上限1000字），短内容更易传播
- **格式**：3-5张图片卡片（封面+步骤卡）
- **文案**：1句钩子 + 3-5个要点 + CTA
- **标签**：文末5-10个标签
- **语调**：友好、个人化，像推荐给朋友
- **CES权重提醒**：评论=点赞×4，关注=点赞×8

### 即刻
- **字数**：不超过200字
- **格式**：一句金句或一个令人惊讶的数据点
- **标签**：2-3个相关标签
- **语调**：随意、对话式，不需要格式
- **目标**：引发评论区讨论，不是传递完整内容

### X / Twitter
- **格式**：最多5条推文线程
- **字数**：免费账号每条≤280字符；Premium账号每条≤500字符
- **风格**：emoji + hashtags；第一条必须是钩子
- **线程流程**：钩子 → 问题 → 解决方案 → 证明 → CTA/链接
- **媒体**：第一条推文附带1张图片提升触达

### LinkedIn
- **语调**：专业但个人化；"I learned this" 而非 "Companies should"
- **数据**：包含具体数字和成果
- **结构**：短钩子段落 + 3-4个要点 + 开放式问题
- **问题**：以问题结尾驱动评论（LinkedIn算法奖励评论）
- **标签**：3-5个专业标签

### Dev.to
- **Frontmatter**：YAML frontmatter 包含 title, published: true, tags, series
- **代码块**：三反引号 + 语言标签
- **图表**：原生支持Mermaid
- **语调**：友好、包容、开发者对开发者
- **系列**：如果是AI Prism系列，设置 series: "AI Prism"

## 适配工作流

```
1. 以完整中文文章为母版（微信公众号版本）
2. 长文平台（知乎、掘金、Dev.to）：按需裁剪或扩展，保留结构
3. 短内容平台（小红书、即刻、X）：提取最有价值的单一洞察
4. 专业平台（LinkedIn）：重新框架为职业/效率洞察
5. 所有版本在规则允许时包含回链到完整文章
```

## 发布检查清单

### 通用检查
- [ ] 标题符合目标平台公式？
- [ ] 开头3秒内有钩子？
- [ ] 字数在平台推荐范围内？
- [ ] CTA清晰且合适？
- [ ] 标签/话题已添加？

### 平台特定检查
- [ ] 微信：段落不超过3行？图片已准备？
- [ ] 小红书：有封面图建议？标签5-10个？
- [ ] X/Twitter：每条<280字符？第一条有图片？
- [ ] LinkedIn：结尾有问题驱动评论？
- [ ] 掘金：有可运行代码？有前言？

## 发布后首小时运营SOP

发布后的首小时是算法推荐的关键窗口期：

**0-15分钟：**
- [ ] 自己先阅读一遍，检查是否有格式错误
- [ ] 分享到3-5个相关社群（不要刷屏）
- [ ] 在评论区先发一条引导性评论（提问或补充）

**15-30分钟：**
- [ ] 回复前3条评论（算法会奖励早期互动）
- [ ] 观察阅读量增长趋势，如果明显低于预期，检查是否被限流

**30-60分钟：**
- [ ] 继续回复评论，保持互动
- [ ] 如果数据好，准备第二波分享
- [ ] 记录首小时数据作为后续优化参考

## 使用方式

- **单平台适配**：说"把这篇文章改成小红书版本"
- **全平台适配**：说"帮我适配所有平台"
- **批量发布准备**：说"准备这周要发布的内容"
- **发布时间建议**：说"什么时候发最好"

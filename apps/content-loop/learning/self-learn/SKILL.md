# Self-Learn — 自学习引擎

让 ContentLoop 像人一样学习，越用越懂你。

## 核心机制：记忆 + 反馈 + 进化

参考 Mem0 和 Reflexion 架构，设计三层学习体系：

```
用户交互 → 记忆提取 → 执行决策 → 效果追踪 → 反馈分析 → 记忆更新
    ↑                                                          │
    └──────────────────── 下次更优执行 ←────────────────────────┘
```

## 记忆系统（三层）

### 第一层：短期记忆（Session Memory）
- **内容**：当前对话上下文、临时偏好、本次任务目标
- **存储**：会话内存
- **生命周期**：单次会话

### 第二层：中期记忆（User Memory）
- **内容**：用户风格偏好、常用表达、历史决策、效果数据
- **存储**：~/.contentos/memory/user-{id}.md
- **生命周期**：持续更新，跨会话保留

```markdown
## 用户记忆档案

### 写作风格
- 句式偏好：[短句为主/长短交替/正式/口语化]
- 常用表达：[口头禅/固定句式/标志性表达]
- 语气特征：[幽默/严肃/温暖/犀利]
- 段落习惯：[平均长度/是否喜欢用列表]

### 历史决策偏好
- 标题偏好：[数字型占比X% / 悬念型占比Y%]
- 结构偏好：[6段式使用X次 / 故事线使用Y次]
- 平台偏好：[公众号X篇 / 小红书Y篇 / 知乎Z篇]
- 发布时间偏好：[周二20:00成功率最高]

### 效果数据
- 最佳标题公式：[公式] — 平均点击率[X%]
- 最佳内容结构：[结构] — 平均完读率[X%]
- 最佳发布时间：[时段] — 平均阅读量[X]
- 最佳情绪切入点：[元素] — 平均互动率[X%]

### 素材库
- 金句库：[列表]
- 案例库：[列表]
- 数据点库：[列表]
- 个人经历库：[列表]

### 学习记录
- [日期]：用户修改了[内容] → 学习到[偏好]
- [日期]：用户选择了[选项A]而非[选项B] → 学习到[偏好]
- [日期]：文章数据[好/差] → 原因分析[结论]
```

### 第三层：长期记忆（System Memory）
- **内容**：跨用户的通用规律、平台规则变化、行业趋势
- **存储**：~/.contentos/memory/system.md
- **生命周期**：持续积累，定期人工审核

## 学习信号

系统从以下信号中学习：

| 信号类型 | 示例 | 学习动作 |
|---------|------|---------|
| **显式反馈** | 用户说"这个标题不好" | 记录标题偏好，降低类似推荐 |
| **隐式反馈** | 用户选择了候选A而非B | 记录选择偏好 |
| **编辑行为** | 用户修改了生成的内容 | 分析修改模式，提取风格特征 |
| **效果数据** | 文章阅读量高/低 | 分析成功/失败因素 |
| **互动数据** | 评论区高频问题 | 更新选题偏好 |
| **时间偏好** | 用户总在晚上创作 | 调整默认交互时间 |

## 进化机制

### 进化1：写作风格学习
```
初始：使用通用写作风格
  → 记录用户每次编辑（改哪里、怎么改）
  → LLM分析编辑模式，提取风格特征
  → 生成时自动匹配风格特征
  → 评估：用户返工率降低？→ 学习有效
```

### 进化2：标题公式学习
```
记录所有生成标题 + 用户选择 + 最终效果
  → 分析高表现标题的共同结构
  → 更新标题公式库，标注每个公式的平均表现
  → 优先推荐高表现公式
  → 定期尝试新公式（探索vs利用）
```

### 进化3：发布时间学习
```
记录每次发布时间 + 效果数据
  → 建立时间-效果映射表
  → 识别最优发布时段
  → 预测新内容的最佳发布时间
  → 随受众变化动态调整
```

### 进化4：选题偏好学习
```
记录用户接受/拒绝的选题推荐
  → 构建用户兴趣向量
  → 预测新话题的相关性
  → 主动推荐高相关性话题
  → 检测兴趣漂移，自动调整
```

## 主动学习（Active Learning）

当系统不确定时，主动询问用户：

```
系统对某话题的相关性不确定（置信度0.6）
  → 向用户展示话题摘要
  → 用户标记：相关 / 不相关 / 稍后关注
  → 更新用户兴趣模型
  → 下次类似话题置信度提升
```

**最小化打扰原则**：
- 每天最多主动询问3次
- 优先在对话自然结束时询问
- 提供"批量确认"选项（一次确认多个）

## 记忆更新规则

参考 Mem0 的两阶段流水线：

### 提取阶段
从最新交互中提取候选记忆事实：
- 用户明确说的偏好（"我喜欢短句"）
- 用户的行为模式（总是选择数字型标题）
- 效果的因果推断（周二发布效果好）

### 更新阶段
判断对现有记忆的操作：
- **ADD**：新增记忆（用户首次表达某偏好）
- **UPDATE**：更新记忆（用户偏好发生变化）
- **DELETE**：删除记忆（记忆已过时或错误）
- **NOOP**：无操作（与现有记忆一致）

## 可解释性

每次进化决策都记录原因：

```markdown
## 进化记录：[日期]

### 变化
- 标题偏好：数字冲击型权重从0.3提升到0.5

### 原因
- 过去5篇使用数字冲击型标题的文章，平均点击率比悬念型高32%
- 用户在过去3次选择中，2次选择了数字冲击型

### 证据
- [文章1] 数字型标题，点击率12%（高于平均8%）
- [文章2] 数字型标题，点击率15%（高于平均8%）
- [用户选择记录] 2026-06-10 选择了"3个方法..."
```

## 安全边界

- **进化上限**：单维度权重变化不超过20%/次，防止剧烈波动
- **负面循环检测**：如果连续3次效果变差，回滚到上一版本
- **用户否决权**：用户可以随时查看、编辑、删除记忆内容
- **定期审核**：每月生成记忆报告，用户确认或修正

# Trending Discover — 热点发现与选题挖掘

从多维度发现可创作的话题，将"不知道写什么"变成"选题太多写不过来"。

## 热点发现渠道

### 实时热点
| 渠道 | 方法 | 适用领域 |
|------|------|---------|
| 微博热搜 | WebSearch "微博热搜 今日" | 大众话题、社会热点 |
| 知乎热榜 | WebSearch "知乎热榜" | 深度讨论、专业话题 |
| GitHub Trending | WebSearch "GitHub trending today" | 技术前沿、开源动态 |
| Product Hunt | WebSearch "Product Hunt popular today" | 产品、创业、AI工具 |
| Hacker News | WebSearch "Hacker News top stories" | 技术深度、创业 |
| 小红书热点 | WebSearch "小红书 热门话题" | 生活方式、消费、美妆 |
| 抖音热点 | WebSearch "抖音 热点榜" | 短视频、大众情绪 |
| 36氪/虎嗅 | WebSearch "36氪 热门" | 科技商业、行业动态 |

**扫描优先级：** 默认先扫描前3个（微博+知乎+GitHub），其他按需搜索。避免一次调用过多WebSearch导致超时。

### 个人IP热点（长期选题库）
- **读者评论区高频问题** → 直接回答就是选题
- **同行文章的评论区** → 他们没回答好的问题
- **自己的实操经验** → "我如何用XX实现YY"
- **行业认知差** → "大多数人不知道的XX"
- **个人成长节点** → 转行、升职、踩坑、复盘

## 选题评估框架

对每个发现的热点，快速评估：

```
热度分(1-10): 当前讨论量有多大？
切入角度: 我能提供什么独特视角？
时效窗口: 还能热多久？(即时/3天/1周/长期)
受众匹配: 是否符合我的读者画像？
内容储备: 我是否有足够的素材/经验支撑？
传播潜力: 是否容易引发讨论/转发？
```

**优先创作条件：** 热度≥7 + 有独特角度 + 时效窗口≥3天

## 热点红线 — 这些热点不要追

| 类型 | 示例 | 原因 |
|------|------|------|
| 政治敏感 | 政策解读、领导人相关 | 风险极高，可能封号 |
| 社会悲剧 | 灾难、事故、刑事案件 | 消费苦难，舆论反噬 |
| 未经证实 | 谣言、爆料、八卦 | 可能传播虚假信息 |
| 过度争议 | 性别对立、地域歧视 | 引战内容，平台限流 |
| 他人隐私 | 个人八卦、内部消息 | 法律风险 |

**判断标准：** 如果追这个热点会让你感到不安，那就不要追。

## 选题输出格式

每次热点扫描后，输出结构化选题清单：

```markdown
## 今日选题推荐 (YYYY-MM-DD)

### 🔥 即时热点（24h内）
1. **[话题]** — 切入角度：[你的独特视角] | 平台建议：[最适合的平台] | 时效：[剩余窗口]
2. ...

### 📊 持续热点（3-7天）
1. **[话题]** — 切入角度：[你的独特视角] | 平台建议：[最适合的平台] | 时效：[剩余窗口]

### 💡 长期选题（随时可写）
1. **[话题]** — 切入角度：[你的独特视角] | 平台建议：[最适合的平台]

### 🎯 推荐优先创作
**首选：[话题]** — 理由：[为什么这个最值得写]
```

## 使用方式

- **每日扫描**：说"扫一下今天的热点"或"找选题"
- **定向搜索**：说"找找AI领域最近有什么热点"
- **灵感模式**：说"我没什么灵感，帮我头脑风暴一下选题"
- **竞品监控**：说"看看同行最近在写什么"（需提供对标账号/博客）

## 冷启动选题指南（无历史内容时）

如果你还没有发布过内容，从以下方向开始：
1. **"我如何做到XX"** — 分享你已经达成的具体成果
2. **"大多数人不知道的XX"** — 行业认知差
3. **"XX工具/方法的完整教程"** — 实用价值
4. **"我在XX上踩过的坑"** — 经验复盘
5. **"XX领域的5个趋势"** — 行业观察

避开：纯理论、没有个人经验、过于宽泛的话题

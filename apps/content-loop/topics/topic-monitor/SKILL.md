# Topic Monitor — 话题监测与追踪

像雷达一样持续扫描你关心的领域，有新动静立刻提醒。

## 两种监测模式

### 模式A：用户主动添加（显式）
用户明确说"帮我监测XX话题"、"追踪这个作者"。

### 模式B：系统主动发现（隐式）
系统根据历史创作内容、用户互动行为，自动推断用户可能感兴趣的新话题，主动推荐监测。

## 监测对象类型

### 1. 话题/关键词监测
监测全网关于特定话题的新内容：
- **监测源**：微博热搜、知乎热榜、小红书热门、抖音热点、公众号文章、掘金/知乎技术文章、GitHub Trending、Product Hunt、Hacker News、arXiv论文、36氪/虎嗅等
- **监测频率**：实时/每小时/每日/每周
- **触发条件**：新内容出现 / 热度超过阈值 / 情感倾向变化

### 2. 链接监测
监测特定URL的内容更新：
- **适用场景**：竞品博客、行业报告页面、政策文件页面、产品更新日志
- **监测方式**：定期抓取对比内容变化（参考 changedetection.io 原理）
- **变化类型**：新增内容 / 内容修改 / 页面删除

### 3. 作者监测
监测特定作者的全平台动态：
- **监测维度**：新文章发布、互动数据变化、粉丝增长、内容风格变化
- **支持平台**：微信公众号、知乎、小红书、即刻、X/Twitter、掘金、GitHub、B站、抖音
- **分析输出**：作者内容节奏、热门话题偏好、互动策略、内容质量趋势

### 4. 渠道监测
监测某个平台/领域的整体趋势：
- **示例**："监测AI领域公众号的整体趋势"、"追踪小红书上护肤类内容的变化"
- **输出**：领域热度趋势、新兴话题、头部内容特征、内容空白机会

## 主动学习机制

系统根据以下信号自动学习和扩展监测范围：

### 学习信号
| 信号类型 | 示例 | 学习动作 |
|---------|------|---------|
| 用户点击 | 用户经常点击AI相关选题 | 增加AI子领域监测 |
| 用户忽略 | 用户从不点击娱乐热点 | 降低娱乐类推荐权重 |
| 创作历史 | 用户写过5篇职场文章 | 增加职场细分领域监测 |
| 评论反馈 | 评论区高频出现"求更新XX" | 将XX加入监测列表 |
| 互动行为 | 用户经常转发某类内容 | 增加同类内容监测 |

### 主动推荐流程
```
系统发现新话题候选
  → 计算与用户历史兴趣的相关性分数
  → 高确定性（>0.8）：直接加入监测，通知用户
  → 中确定性（0.5-0.8）：推荐给用户确认
  → 低确定性（<0.5）：暂不处理，等待更多信号
```

## 监测配置格式

```markdown
## 监测配置：[主题名称]

### 基本信息
- 创建时间：[日期]
- 监测类型：[话题/链接/作者/渠道]
- 监测频率：[实时/每小时/每日/每周]

### 监测源
- [ ] 微博热搜
- [ ] 知乎热榜
- [ ] 小红书热门
- [ ] 公众号（关键词搜索）
- [ ] 掘金/知乎技术区
- [ ] GitHub Trending
- [ ] Product Hunt
- [ ] Hacker News
- [ ] arXiv
- [ ] 其他：[自定义]

### 关键词
主关键词：[核心词]
扩展关键词：[同义词/相关词]
排除关键词：[不相关词]

### 触发条件
- [ ] 有新内容即通知
- [ ] 热度超过[N]时通知
- [ ] 情感倾向为[正面/负面]时通知
- [ ] 仅在[时间段]内通知

### 输出格式
- [ ] 摘要推送（标题+摘要+链接）
- [ ] 详细报告（含分析+建议）
- [ ] 仅记录不通知
```

## 输出示例

```markdown
## 话题监测报告：[主题] — [日期]

### 新发现内容（[N]条）
1. **[标题]** — [来源] — [作者]
   - 摘要：[一句话摘要]
   - 热度：[高/中/低]
   - 与你的相关性：[高/中/低]
   - 建议动作：[选题/参考/忽略]

### 趋势变化
- [话题A] 热度上升 [X]%（原因分析）
- [话题B] 热度下降 [X]%（原因分析）

### 监测到的作者动态
- [作者A] 发布了[内容类型]：[标题] — [亮点分析]

### 系统推荐的新监测方向
基于你的历史兴趣，建议增加监测：
1. [新方向] — 理由：[分析]
```

## 与 content-os 的协作

topic-monitor 的发现直接输入到：
- **trending-discover**：作为热点发现的补充数据源
- **topic-analyzer**：监测到的新内容可直接进入选题分析
- **content-os**：高相关性内容自动推送给用户作为选题建议
```

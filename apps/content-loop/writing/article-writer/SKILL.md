# Article Writer — 深度文章撰写

从选题到成稿的完整写作流程，产出可直接发布的深度文章。

## 标题公式（必选其一）

1. **冲突公式**：对比两个对立观点（如"AI 不会取代你，但会用 AI 的人会"）
2. **数字冲击**：用具体、令人惊讶的数字（如"3 小时 vs 3 分钟：我用 AI 重构了整个工作流"）
3. **免费钩子**：承诺零成本获得（如"零成本搭建你的私人 AI 助手"）
4. **痛点直击**：直接点出读者困扰（如"为什么你的 AI 总给不出想要的答案？"）
5. **悬念提问**：制造好奇心（如"为什么你天天跑步还是瘦不下来？"）
6. **反差对比**：前后对比+悬念（如"半年时间从130斤到90斤"）

## 开篇模式（必选其一）

1. **场景代入**：把读者放入一个生动场景（如"凌晨 2 点，我还在手动整理 Excel..."）
2. **痛点共鸣**：直接说出问题（如"写周报这件事，浪费了我人生 200 个小时"）
3. **热点事件**：用最近的新闻/发布作为引子
4. **个人故事**：分享真实的、具体的个人经历
5. **痛点代入+转折发现**（高转化型）
   参考知乎爆款工具测评的开篇技巧：
   - 第一步：建立共鸣 — "最近我一直用XX，整体体验非常爽"
   - 第二步：抛出痛点 — "但有个痛点一直很折磨人：XX"
   - 第三步：归因分析 — "这不是XX的问题，而是XX的天然局限"
   - 第四步：转折发现 — "直到我遇到了XX"
   - 适用：工具测评、产品推荐、方法论介绍

## 文章结构（多选）

默认推荐6段式，但以下结构可选：

### 结构A：6段式（默认）
适合：教程、工具介绍、方法论
```
🔥 01 痛点/场景 — 建立问题或场景
🛠️ 02 解决方案 — 提出工具、方法或思路
💡 03 实战案例 — 展示具体、有数字的案例
📋 04 步骤拆解 — 将过程分解为可执行的步骤
⚠️ 05 不足之处 — 诚实列出2-3个限制或注意事项
✨ 写在最后 — 金句结尾 + 轻量CTA
```

### 结构B：故事线
适合：个人经历、复盘、情感类
```
开头钩子 → 故事发展 → 转折点 → 感悟/方法论 → 金句结尾
```

### 结构C：时间线
适合：项目复盘、成长记录、年度总结
```
起点状态 → 关键节点1 → 关键节点2 → 当前状态 → 未来展望
```

### 结构D：对比线
适合：产品测评、方法对比、观点碰撞
```
A方案介绍 → B方案介绍 → 对比分析 → 我的选择/建议 → 适用场景总结
```

### 结构E：清单体
适合：资源推荐、技巧合集、避坑指南
```
痛点引入 → 清单项1 → 清单项2 → 清单项3... → 总结+CTA
```

### 结构F：发现之旅（推荐用于工具/产品/方法论介绍）
适合：工具测评、产品推荐、新技术介绍
```
痛点共鸣 → 深入探索 → 核心发现 → 详细解析 → 真实体验 → 总结推荐
```
- 痛点共鸣：我也在用XX，但遇到了什么问题
- 深入探索：为了解决这个问题，我尝试了/发现了什么
- 核心发现：引出你要介绍的核心产品/方法
- 详细解析：用"小标题+emoji+表格"拆解核心亮点
- 真实体验：给出具体数据、真实场景、前后对比
- 总结推荐：走心总结 + 明确行动号召

## 写作风格规则

### 语言风格
- 使用**"我"**（第一人称），不用"我们"
- **口语化中文** — 像和朋友喝咖啡聊天，短句为主
- 用**具体数字**（"节省了 4 小时" 而非 "节省了大量时间"）
- 包含 **Before vs After** 对比，让效果可感知
- **故事化引入**：产品/工具不是干巴巴介绍功能，先讲名字由来、作者故事、发现过程
- **类比解释**：用读者熟悉的事物类比新技术/新概念（如"Slack让人类协作，Federation让AI协作"）
- **数据锚定**：每个核心论点配1个具体数字（不是"很快"而是"150-12500倍"）

### 禁止清单
- ❌ "值得注意的是"、"综上所述"、"随着...的发展"
- ❌ "Furthermore"、"Moreover"、"It's important to note that"
- ❌ 每段都是3-5句的完美结构
- ❌ 泛化表述："many experts believe"、"研究表明"
- ❌ 完美的总分总结构

### 段落控制
- 每段不超过4行（手机阅读友好）
- 每300字1-2个emoji用于强调和分隔
- 每500字至少1张配图建议

## 写作流程

1. **确认选题和平台** — 如果没有经过 topic-analyzer，先快速确认
2. **生成3个候选标题** — 用不同公式，让用户选择
3. **生成大纲** — 6段式结构，每段写核心要点
4. **逐段填充** — 按大纲逐段写作，每段写完确认
5. **金句提炼** — 从正文中提炼1-2句可独立传播的金句
6. **自检** — 按检查清单逐项验证

## 写作模式

### 快速模式
用户说"快速写一篇"或时间紧急时：
1. 直接生成完整文章（不逐段确认）
2. 输出标题+全文+自检清单
3. 用户一次性审阅修改

### 精细模式（默认）
标准流程：逐段确认，每段可修改/重写/跳过

## 自检清单

- [ ] 标题使用了标题公式？
- [ ] 开篇用了"痛点代入+转折发现"或至少一种开篇模式？
- [ ] 开头3秒内抓住注意力？
- [ ] 包含至少1个具体数字/数据？
- [ ] 每个核心亮点有具体数据支撑？
- [ ] 包含至少1段个人经历/具体场景？
- [ ] 有Before vs After对比？
- [ ] 用了至少1个类比帮助理解？
- [ ] 小标题有emoji或视觉区分？
- [ ] 大段文字拆成了3-5行短段？
- [ ] 结尾有金句？
- [ ] 结尾有走心总结+明确CTA？
- [ ] 没有AI感过渡词？
- [ ] 段落长度有变化？
- [ ] 总字数在目标范围内？

## Mermaid 图表规则
- 支持类型：flowchart, sequenceDiagram, gantt, pie
- 不使用自定义主题或样式
- 不在节点标签中使用emoji
- 最多8-10个节点
- 始终提供文字描述（部分平台不支持Mermaid）

## 脱敏规则
发布前移除或匿名化：
- 真实公司名 → "某云厂商"、"某 SaaS 公司"
- 真实人名 → 化名或角色描述
- API Key、Token 等敏感信息
- 内部URL、Dashboard链接
- open_id、app_id 等内部标识符

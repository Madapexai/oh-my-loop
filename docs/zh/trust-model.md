# 可信模型

Oh My Loop 是 alpha 阶段的方法论与参考实现。它减少可预见的循环失败，但不能让模型输出天然变得真实、安全或适合每个人。

## 必须保持的性质

- 先判断风险与权限，再判断任务是否简单。
- 路由模型负责识别紧急人身风险；一旦判为 critical，不再调用执行或优化模型，立即停止自动化并寻求人类支持。
- 医疗、法律、重大财务、职业、关系等人生决策中，用户始终是决策者。
- 对外、不可逆、昂贵、涉及隐私或影响他人的动作，需要针对“这一次具体动作”的确认。
- `completed` 必须有新鲜、相关、可追溯的证据，并通过副作用检查。
- 空验证器、验证异常、模糊的“完成”表述均按未完成处理。
- 每个循环都有次数、时间、成本、停滞和取消边界。
- 记忆能力默认开启；个人内容持久化仍需同意，新条目默认隔离，并具有来源、期限、纠正和遗忘能力。

## 为什么仍然可能失败

模型语义路由仍可能误解隐喻、文化语境、歧义或对抗性表达；验证者也可能与执行者犯同一种错误；观测可能陈旧；代理指标可能变好而真实生活变差；记忆可能被污染；用户也可能逐渐依赖系统、失去自主性。因此，不确定性必须降低自动化权限，而不是增加权限。项目不存在规则式语义回退：缺少模型、调用失败或结构化输出无效时一律安全失败。

## 能力边界

本地 JSON 运行记录和记忆未加密，不适合秘密或高度敏感信息。CLI 是持久化控制面，不是隔离的任意工具执行引擎；动作仍由用户或宿主 Agent 执行。哈希链只能帮助发现修改，不能阻止有本机文件权限的人重写整条链。Agent Team 的不同角色若共享模型、供应商或证据，其错误也可能相关。项目不是急救、医疗、心理、法律或金融服务。生活 Loop 的正确目标通常是帮助用户看清选择、做小而可逆的实验，也允许休息、接受现状或结束优化。

## 如何理解评测分数

本地路由评测只是内部回归证据，不是独立的真实世界效果证明，生成的评测结果不提交。可信评测需要按风险、语言和人群分别报告召回率与误报率，并增加端到端任务、伤害指标、长期结果、红队测试、事故复盘和外部审查。不能用一个总体准确率掩盖关键风险类别的失败。

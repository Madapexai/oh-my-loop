# 成本节约规范

## 🎯 核心理念

> "省钱为基础" -- team-sop核心原则
> "优先找免费资源，但有隐性成本" -- Yason 5-30
> "烧token厉害" -- Agent Team复盘报告解散原因之一

省钱不是抠门，是**警惕隐性成本+优先免费+避免浪费**。

## 🪙 Token节约原则（类似token juice）

### 模型选择优先级
1. **免费模型优先** - DeepSeek/Kimi/智谱等免费额度
2. **便宜模型兜底** - 简单任务不用贵模型
3. **贵模型只在必要时** - 复杂推理才用Claude/GPT-4

### 调用节约技巧
- **Prompt精简** - 不废话，直奔主题
- **批量处理** - 一次处理多条，不逐条调
- **缓存复用** - 相同输入缓存结果
- **截断输出** - 不需要的内容及时截断
- **流式中断** - 发现跑偏立刻中断

### 上下文管理
- **定期压缩** - 长对话定期总结压缩
- **清理无关** - 不相关的上下文及时清理
- **子代理隔离** - 大输出用子代理，不污染主上下文

## ⚠️ 隐性成本识别

> "优先找免费资源，但有隐性成本" -- Yason

| 免费资源 | 可能的隐性成本 |
|---------|--------------|
| Google Colab | 会话断开、资源不稳定 |
| 免费模型额度 | 速率限制、数据被训练 |
| 开源工具 | 维护成本、学习曲线 |
| 自建服务 | 服务器费、电费、时间 |
| 免费API | 突然限流、停服风险 |

### 决策原则
- **短期试水** - 免费/便宜优先
- **长期生产** - 算TCO（总拥有成本）
- **关键路径** - 不省钱，用稳定可靠的
- **非关键** - 能免费就免费

## 🔍 成本审查检查点

### 调用API前
- [ ] 必须用这个模型吗？便宜的不行吗？
- [ ] Prompt精简了吗？
- [ ] 能批量吗？
- [ ] 能缓存吗？

### 使用云资源前
- [ ] 有免费替代吗？
- [ ] 用量在免费额度内吗？
- [ ] 超额后怎么收费？
- [ ] 长期用还是临时用？

### 技术选型时
- [ ] 开源方案行不行？
- [ ] 自建 vs 订阅，TCO谁低？
- [ ] 有锁定风险吗？
- [ ] 迁移成本高吗？

## 🚫 烧token的常见反模式

| 反模式 | 问题 | 改进 |
|--------|------|------|
| 重复读大文件 | 每次消耗token | 读一次，记住关键 |
| 长上下文不压缩 | 越聊越贵 | 定期总结压缩 |
| 逐条调API | 不批量 | 批量处理 |
| 用贵模型做简单任务 | 杀鸡用牛刀 | 模型匹配任务 |
| 不截断输出 | 浪费输出token | 及时截断 |
| 反复试错 | 没根因调查 | 系统调试一次过 |

## 📊 成本看板（参考团队模式）

> 模型路由统一走 mico，mico 内置成本看板。

### 关键指标
- 日均token消耗
- 日均费用
- 每任务平均成本
- 免费vs付费比例
- 异常消耗告警

### Fallback策略（mico配置）
```
免费模型(DeepSeek) -> 备用免费(Kimi) -> 便宜付费(DeepSeek付费) -> 贵模型(Claude)
```

```
  |
  v

  |NO
  v

  |NO
  v
必须用贵模型吗? --YES--> 用,但精简prompt
  |NO
  v
重新评估需求,是否过度设计
```

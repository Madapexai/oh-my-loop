# Self-Questioning

Most wrong answers come from stopping at the first plausible option. Self-questioning forces you to consider alternatives.

## When to use

- High-stakes decisions (irreversible, expensive, affects users)
- When the first answer feels too easy
- When you haven't explored alternatives

## When NOT to use

- Trivial tasks -> just answer
- Time-critical -> decide and act
- You've already questioned enough -> move on

## The question set

Before committing to a plan or answer, ask:

1. **Is this the only option?** What are 2-3 alternatives?
2. **What would fail?** List 3 failure modes.
3. **What am I assuming?** List assumptions; verify the riskiest one.
4. **What's the cheapest way to test this?** Before full commitment.
5. **If I'm wrong, what's the cost?** Is it acceptable?

## Multi-perspective check

For high-stakes, run 3 perspectives:

- **Optimist** - why this plan works
- **Pessimist** - why it fails
- **Judge** - which is more likely, what's the compromise

## Anti-patterns

- **Questioning forever** - never deciding. Set a max of 1-2 rounds.
- **Questioning trivia** - don't question "what's 2+2"
- **Questioning to justify** - asking questions whose answers support your preferred option

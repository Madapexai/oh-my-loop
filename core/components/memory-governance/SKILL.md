# Memory Governance

Memory is a claim with a lifecycle, not a transcript dump.

Each entry needs source, timestamp, sensitivity, confidence, expiry, status, and version. Personal or sensitive memory requires explicit consent. New entries start quarantined and cannot influence future action until reviewed or otherwise validated.

Retrieval must respect purpose and scope. Show the user which memory affected a recommendation. Support correction, supersession, rejection, export, and forgetting. Expired or corrupted memory fails closed. Do not silently replace an unreadable store with an empty one.

Never store secrets, crisis disclosures, inferred diagnoses, or judgments about another person merely because they may be useful later.
